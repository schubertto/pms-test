# 팀 프로젝트 진행상황 관리 시스템 (PMS) 제품 요구사항 정의서 (PRD)

본 문서는 애자일(Agile) 개발 방식에 맞추어 팀 프로젝트의 진행상황, 일정(갠트 차트), WBS(작업 분할 구조), 그리고 팀원별 진척율을 관리하기 위한 시스템의 기획서 및 데이터베이스(Neon DB) 설계 문서입니다.

---

## 1. 서비스 개요 🌐
* **목적**: 프로젝트의 WBS(업무 분할 구조)와 타임라인(갠트 차트), 그리고 팀원별 역할에 따른 시작일/종료일 및 진척율을 효율적으로 시각화하고 관리합니다.
* **일상적 비유**: 
  * 프로젝트는 **'한 대의 함선'**이고, 팀원들은 **'각자의 구역을 맡은 선원들'**입니다.
  * 본 시스템은 함선의 현재 속도(진척율), 도착 일정(갠트 차트), 그리고 각 선원들의 업무 교대 시간(시작일/종료일)을 보여주는 **'선장실의 관제 모니터'** 역할을 합니다.

---

## 2. 팀원 및 역할 정의 👥
팀원들의 역할은 고유하며, 한 팀원이 여러 직무를 병행하거나 각자 고유한 파트를 담당합니다.
1. **팀원 1**: 기획자 (Planner) & 테스터 (Tester)
2. **팀원 2**: 백엔드 개발자 (Backend Developer)
3. **팀원 3**: 프론트엔드 개발자 (Frontend Developer)
4. **팀원 4**: API 개발자 (API Developer)
5. **팀원 5**: 디자인 및 퍼블리셔 (Designer & Publisher)

---

## 3. 주요 기능 요구사항 명세 (Functional Requirements)

### 3.1. 대시보드 및 전체/팀원별 진척율 관리 (Dashboard) 📊
* **전체 진척율**: 프로젝트에 등록된 모든 작업의 진행상황을 가중치(작업 난이도) 기준으로 자동 계산하여 대시보드 메인에 게이지 차트로 출력합니다.
* **팀원별 진척율**: 각 팀원이 맡은 작업들의 진행 상태를 계산하여 팀원별 기여도 및 진척율을 막대그래프로 비교 시각화합니다.
* **스프린트 현황**: 현재 활성화된 애자일 스프린트의 남은 기간과 핵심 목표(마일스톤)를 보여줍니다.

### 3.2. WBS 구조 및 갠트 차트 (Gantt Chart) 📅
* **WBS 트리 구조**: 작업을 계층형(예: 1. 기획 -> 1.1 UI 기획, 1.2 DB 설계)으로 구성하여 체계적으로 관리합니다.
* **갠트 차트 타임라인**: 
  * 마일스톤 및 작업별 시작일과 종료일을 시각적인 막대그래프로 표시합니다.
  * 마우스를 올리면 상세 정보(담당자, 상태, 진척율)가 툴팁으로 표시됩니다.
* **작업 간 의존성(Dependency) 시각화**: A 작업이 끝나야 B 작업을 할 수 있는 관계를 시각적으로 가이드합니다.

### 3.3. 애자일 칸반 보드 (Kanban Board) 📋
* **상태 전환**: 작업을 4단계 상태로 관리합니다.
  * **예정** / **진행중** / **완료** / **취소**
* **드래그 앤 드롭 또는 원클릭 상태 전환**: 클릭이나 드래그를 통해 작업 상태를 쉽게 변경하고 DB에 즉각 반영합니다.

### 3.4. 작업 및 일정 관리 (CRUD) ✏️
* **추가(Create)**: 신규 작업을 WBS 코드, 스프린트, 담당 팀원, 시작/종료일, 가중치(스토리 포인트)와 함께 등록합니다.
* **조회(Read)**: 스프린트 필터, 팀원 필터, 상태 필터를 통해 원하는 작업만 정밀 필터링합니다.
* **수정(Update)**: 일정 변경, 진척율 업데이트, 담당자 변경 등을 수행합니다.
* **삭제(Delete)**: 작업 삭제 시 확인 경고를 띄워 실수를 방지합니다.

## 4. 데이터베이스 구조 설계 (Neon DB Schema) 💾

PostgreSQL(Neon DB)을 사용하여 아래와 같이 테이블 구조를 형성합니다.

### 4.1. 팀원 테이블 (`members`)
| 컬럼명 | 데이터 타입 | 제약 조건 | 설명 |
| :--- | :--- | :--- | :--- |
| `id` | SERIAL | PRIMARY KEY | 팀원 고유 ID |
| `name` | VARCHAR(50) | NOT NULL | 팀원 이름 (예: 팀원1, 팀원2 등) |
| `role` | VARCHAR(100) | NOT NULL | 주 역할 설명 (예: 기획자 및 테스터) |
| `start_date` | DATE | NOT NULL | 프로젝트 참여 시작일 |
| `end_date` | DATE | NOT NULL | 프로젝트 참여 종료일 |

### 4.2. 스프린트 테이블 (`sprints`)
| 컬럼명 | 데이터 타입 | 제약 조건 | 설명 |
| :--- | :--- | :--- | :--- |
| `id` | SERIAL | PRIMARY KEY | 스프린트 고유 ID |
| `name` | VARCHAR(100) | NOT NULL | 스프린트 이름 (예: 스프린트 1) |
| `start_date` | DATE | NOT NULL | 스프린트 시작일 |
| `end_date` | DATE | NOT NULL | 스프린트 종료일 |
| `is_active` | BOOLEAN | DEFAULT FALSE | 현재 활성화 여부 |

### 4.3. 작업 테이블 (`tasks`)
| 컬럼명 | 데이터 타입 | 제약 조건 | 설명 |
| :--- | :--- | :--- | :--- |
| `id` | SERIAL | PRIMARY KEY | 작업 고유 ID |
| `title` | VARCHAR(255) | NOT NULL | 작업명 |
| `description` | TEXT | | 작업 상세 내용 |
| `wbs_code` | VARCHAR(50) | NOT NULL | WBS 코드 (예: `1.1`, `2.1.2`) |
| `status` | VARCHAR(20) | CHECK (status IN ('예정', '진행중', '완료', '취소')) | 작업 상태 |
| `weight` | INTEGER | DEFAULT 1 | 가중치 (난이도/스토리 포인트, 1~5) |
| `assignee_id` | INTEGER | REFERENCES members(id) | 담당 팀원 ID |
| `sprint_id` | INTEGER | REFERENCES sprints(id) | 소속 스프린트 ID |
| `start_date` | DATE | NOT NULL | 작업 시작일 |
| `end_date` | DATE | NOT NULL | 작업 종료일 |
| `progress` | INTEGER | CHECK (progress BETWEEN 0 AND 100) | 작업의 진척율 (%) |
| `parent_id` | INTEGER | REFERENCES tasks(id) | 상위 작업 ID (WBS 계층용) |
| `dependency_id` | INTEGER | REFERENCES tasks(id) | 선행 필수 작업 ID |

## 5. 프로젝트 구축을 위한 5단계 로드맵 🚀

1. **[1단계] Neon DB 스키마 생성 및 초기 데이터 주입**:
   * 설계된 스키마에 따라 테이블을 생성하고, 제공된 팀원 5명의 기본 정보 및 테스트용 초기 더미 태스크 데이터를 주입합니다.
2. **[2단계] Streamlit 어플리케이션 및 연결부 구성**:
   * `.env.local` 파일에서 DATABASE_URL을 읽어 Neon DB에 안전하게 접속하는 모듈을 구현합니다.
3. **[3단계] 핵심 대시보드 및 팀원 진척율 시각화**:
   * 전체 진척율, 팀원별 진척율 통계를 계산하고 Plotly 차트로 시각화합니다.
4. **[4단계] WBS 테이블 뷰 및 Plotly Gantt Chart 구현**:
   * WBS 트리 형태의 업무 표와 막대그래프 형태의 갠트 차트를 렌더링하고, 의존성 관계를 표시합니다.
5. **[5단계] 애자일 칸반 보드 및 작업 관리 CRUD 패널 완성**:
   * 작업을 실시간으로 추가/수정/삭제하고 드롭다운을 통해 빠르게 칸반 상태를 이동시키는 인터랙티브 UI를 구현합니다.
